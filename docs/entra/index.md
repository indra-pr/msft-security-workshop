## What Microsoft Entra is

Microsoft Entra secures access for **workforce, customer, partner, workload, and agent (AI) identities** to apps, data, and resources across multicloud and on-premises environments. Built on **Zero Trust** principles, it helps you establish access controls, reduce identity risk, and provide secure access.

## Product family

<div class="grid cards" markdown>

-   :material-account-key:{ .lg .middle } __Microsoft Entra ID__

    ---

    The foundational cloud identity and access management service — authentication, policy enforcement (Conditional Access), and protection for users, devices, apps, and resources (formerly Azure Active Directory).

-   :material-account-multiple:{ .lg .middle } __Microsoft Entra External ID__

    ---

    Identity for **customers and partners** (CIAM) — secure sign-up/sign-in for external users.

-   :material-cog-sync:{ .lg .middle } __Microsoft Entra Workload ID__

    ---

    Identities for **apps, services, and workloads**, with protection and lifecycle controls.

-   :material-shield-account:{ .lg .middle } __Microsoft Entra Suite__

    ---

    ID Protection, ID Governance, Verified ID, Internet Access, and Private Access.

-   :material-server-network:{ .lg .middle } __Microsoft Entra Domain Services__

    ---

    Managed domain services (group policy, LDAP, Kerberos/NTLM) for legacy apps.

-   :material-robot-outline:{ .lg .middle } __Microsoft Entra Agent ID__

    ---

    Purpose-built identity for **AI agents** — authenticate, authorize, govern, and protect nonhuman identities at scale.

</div>
